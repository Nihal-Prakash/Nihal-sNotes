Types of Authentication

  1. Stateful : The Client sends in authentication details, the server verfies and generates a sessionID. It then bundles it and the user details in a HTTP-only cookie and sends it back to the client along side storing it in Redis. With every subsequent request the client sends back the cookie which the server verifies against Redis.

    Pros: Centrailised control because of the persistence store
    Cons: Scalability and higher operational complexity, lantency
  1. Stateless : The Client sends in authentication details, the server verifies and creates a JWT token and signs it with it’s secret key, which it sends back to the client. In the subsequent requests the JWT token is attached in the header which the server can extract and and verify. It is called stateless because all the information is present in the token itself.
  1. API Key : For a progammatic access to a server in a confined manner. useful for machine to machine interactions.
