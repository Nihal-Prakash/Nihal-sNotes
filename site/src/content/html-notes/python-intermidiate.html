Data Class Builders

Three main class builders to write data class with:

  1. collections.namedtuple
  1. typing.NamedTuple requiring type hints on the feilds
  1. @dataclass.dataclass A classs decotator allowing more customization

The Problem 

class Coordinate:

	def __init__(self, lat, lon):
		self.lat = lat
		self.lon = lon

__repr__ inherited is not useful, and __eq__ inherited compares objectIDs. The data class builders covered provide the necessary __init__ ,__repr__ and __eq__ methods automatically.

>>>from coordinates import Coordinate
>>>moscow = Cooordinate(55.76, 37.62)

False

Using namedtuple :

from collections import namedtuple

Coordinate = namedtuple('Coordinate', 'lat lon')
issubclass(Coordinate, tuple)
   
moscow = Coordinate(55.76, 37.617)
moscow

Using typing.NamedTuple :

import typing
Coordinate = typing.NamedTuple('Coordinate', [('lat', float), ('lon', float)])

Using @dataclass :

Signature for @dataclass:

@dataclass(*, init = True, repr = True, eq = True, order = False, unsafe_hash = False, frozen = False)
from dataclasses import dataclass 

@dataclass(frozen==True)
class Coordinates:
    lat: float
    lon: float

    def __str__(self):
        ns = 'N' if self.lat >= 0 else 'S'
        we = 'E' if self.lon >= 0 else 'W'
        return f'{abs(self.lat):.1f}{ns}, {abs(self.lon):.1f}{we}'

Type Hinting: or type annotations are ways to declare the expected type of a function arguements, return values, variables and attributes. It is not enforced by the Python bytecode or the interpreter.

  • Has no Runtime Effect
  • Primarily intended for third party type checkers, more of a static analysis tool
  • var_name: some_type = default_value
  • during module loading time Python reads to build the __annotations__ dictionary
  • Without any value association any feild survives only as annotation
