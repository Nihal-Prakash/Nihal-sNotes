Obj. Reference, Mutability and Recycling

Variables are not boxes , they are labels associated with objects. This can be illustrated with the following: 

>>>a = [1,2,3]
>>>b = a
>>>a.append(4)
>>>b
[1,2,3,4]

Because variables are mere labels, nothing prevents an object from having several labels assigned to it, which is called aliasing.

>>> charles = {'name': 'Charles L. Dodgson', 'born': 1832}
>>> lewis = charles
>>> lewis is charles
True
>>> id(lewis), id(charles)
(140124663838912, 140124663838912)
>>> alex = {'name': 'Charles L. Dodgson', 'born': 1832}
>>> alex == charles
True
>>> alex is charles
False

== compares the values of the object whereas is compares Object Ids

Del and Garbage Collections

Objects are never explicitly destroyed, however, when they become unreachable they may be garbage collected
  1. del is a statement and not a function. There is a special __del__ method which is invoked by the Python interpreter when an instance is about to be destroyed to give it a chance to release external resources.
  1. del deletes refernces not objects, that is for the garbage collector to handle.

def tag(name, /, *content, class_=None, **attrs):
    """Generate one or more HTML tags"""
    if class_ is not None:
        attrs["class"] = class_

    attr_pairs = (
        f' {attr}="{value}"'
        for attr, value in sorted(attrs.items())
    )
    attr_str = "".join(attr_pairs)

    if content:
        elements = (
            f"<{name}{attr_str}>{c}</{name}>"
            for c in content
        )
        return "\n".join(elements)
    else:
        return f"<{name}{attr_str} />"

Decorators

A decorator is a callable that takes another function as an arguement. A decorator may preform some processing with the decorated function, and returns it or replaces it with another function or callable object.

registry = []

def register(func):
    print(f'running register({func})')
    registry.append(func)
    return func

@register
def f1():
    print('running f1()')

@register
def f2():
    print('running f2()')

def f3():
    print('running f3()')

def main():
    print('running main()')
    print('registry->', registry)
    f1()
    f2()
    f3()

if __name__ == '__main__':
    main()

Python variable-scope rules

Python resolves names using LEGB:

  1. L — Local: current function
  1. E — Enclosing: outer functions
  1. G — Global: module-level namespace
  1. B — Built-ins: names such as print, len, range 
x = "global"

def outer():
    x = "enclosing"

    def inner():
        x = "local"
        print(x)   # local

    inner()

The critical rule: assignment makes a name local

Inside a function, if Python sees an assignment to a name anywhere in that function body, it classifies that name as local for the entire function, unless declared global or nonlocal.

b = 6

def f():
    print(b)
    b = 9

This raises:

UnboundLocalError: local variable 'b' referenced before assignment

Even though b = 9 appears after print(b), Python determines scope when compiling the function:

b = 9 exists in function
        ↓
b is classified as local everywhere in f
        ↓
print(b) tries to read local b
        ↓
local b has not yet been assigned

It does not fall back to the global b.

Reading a global variable

Reading requires no declaration:

b = 6

def f():
    print(b)   # reads global b

Modifying a global variable

Use global:

b = 6

def f():
    global b
    print(b)
    b = 9

Now b = 9 updates the module-level variable.

Modifying an enclosing function’s variable

Use nonlocal:

def outer():
    x = 10

    def inner():
        nonlocal x
        x = 20

    inner()
    print(x)   # 20

nonlocal searches enclosing function scopes—not the global scope.

Parameters are local

def f(a):
    print(a)

a belongs to f’s local scope.

Scope versus lifetime

A function’s local variables exist only while that function call is active, except when retained by a closure:

def outer():
    x = 10

    def inner():
        return x

    return inner

Here x survives because inner closes over it.

Bytecode connection

A local variable is generally accessed using optimized local-variable instructions such as:

LOAD_FAST
STORE_FAST

A global name uses instructions such as:

LOAD_GLOBAL
STORE_GLOBAL

That is why Python must classify names when compiling the function body.

Core rule

Reading a name searches outward through LEGB. Assigning a name inside a function makes it local unless global or nonlocal says otherwise.

Closures

A closure is a function that retains access to variables from its enclosing function scope even after the enclosing function has returned.

def make_multiplier(factor):
    def multiply(value):
        return value * factor
    return multiply

double = make_multiplier(2)

print(double(10))

factor is a free variable in multiply. Python stores it inside the returned function's closure.

A nested function is not necessarily a closure. It becomes one only when it references variables from an enclosing function scope.

Late Binding

Closures retain variable bindings, not frozen copies of their values.

functions = [lambda: number for number in range(3)]

print([function() for function in functions])
[2, 2, 2]

All functions reference the same final number.

Capture the current value using a default argument:

functions = [lambda number=number: number for number in range(3)]

print([function() for function in functions])
[0, 1, 2]

The nonlocal Keyword

nonlocal allows a nested function to rebind a variable from the nearest enclosing function scope.

def make_counter():
    count = 0

    def counter():
        nonlocal count
        count += 1
        return count

    return counter

counter = make_counter()

print(counter())
print(counter())

Without nonlocal, assignment makes count local to counter, causing an UnboundLocalError.

nonlocal is required for rebinding:

count += 1

It is not required for mutating an existing object:

def make_collector():
    values = []

    def collect(value):
        values.append(value)
        return values

    return collect

nonlocal variables must already exist in an enclosing function scope. It cannot refer directly to a global variable.

Parameterized Decorators

A parameterized decorator accepts configuration before receiving the function.

from functools import wraps

def repeat(times):
    def decorator(function):
        @wraps(function)
        def wrapper(*args, **kwargs):
            result = None

            for _ in range(times):
                result = function(*args, **kwargs)

            return result

        return wrapper

    return decorator
@repeat(3)
def greet(name):
    print(f"Hello, {name}")

greet("John Doe")

The syntax:

@repeat(3)
def greet():
    pass

is equivalent to:

greet = repeat(3)(greet)

A parameterized decorator normally has three callable levels:

  1. Decorator factory — receives configuration such as times
  1. Decorator — receives the function
  1. Wrapper — receives runtime arguments

Closure relationships:

  • decorator retains times
  • wrapper retains times and function

Decorator arguments are evaluated when the def statement executes. Wrapper arguments are received later when the decorated function is called.

Stacked Decorators

Decorators are applied from bottom to top.

@decorator_a
@decorator_b
def function():
    pass

is equivalent to:

function = decorator_a(decorator_b(function))
