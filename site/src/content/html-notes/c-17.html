Gauranteed Copy Elision

Optimisation method by which the compiler must construct certain temporary objects in their target storage omitting copy/move construction entirely.

This optimisation technique is so important that is works in -O0 also. To stop use the flag -fno-elide-constructors 

#include <iostream>
#include <utility>

struct S {
  int x;

  explicit S(int x) : x{x} { std::cout << "construct S(" << x << ")\n"; }

  S(const S &other) : x{other.x} { std::cout << "copy S(" << x << ")\n"; }
};

S make_value(int x) { return S(x); }

S make_double_value(int x) {
  S s = make_value(x);
  s.x = 2 * x;
  return s;
}

int main(int argc, char *argv[]) {
  S s = make_double_value(21);
  std::cout << "answer: " << s.x << "\n";
  return 0;
}

Updated Value Categories

Realted with Copy Elision, C++17 gave proposed defenitions updates for glvalue and prvalue:

  1. glvalue : expression whose evaluation computes the location of an object, bit-feild, or function
  1. prvalue : expression whose evaluation initialises an object, bit-feild, or function, or operand of an opeartor.

In C++17 Copy Elision/deffered tempoaray Materialization happens when:

  1. in initialisation of an object from a prvalue: Type t = T()
  1. in a function call where the function returns a prvalue

C++98/03 had only lvalue and rvalue, C++11 extended to xvalue, prvalue and glvalue

General Language Features

Init Statement for if and switch

Previously for scoped iterators,

{
  auto val = GetVal();
  if (condition(val)) {
    //success
  } else {
    //failure
  }
}

//example
const std::string myString = "Hello World to C++17";

{
  const auto pos = myString.find("Hello");
  if (pos != std::string::npos) 
    std::cout << pos << "Hello\n";
}

{
  const auto pos = myString.find("World");
  if (pos != std::string::npos) 
    std::cout << pos << "World\n";
}

With C++17,

if (auto val = Getval(); condition(val)) {
  //success 
} else {
  //failure
}

//example 
const std::string myString = "Hello World to C++17";

if (const auto pos = myString.find("Hello"); pos != std::string::npos) {
    std::cout << pos << "Hello\n";
} 
if (const auto pos = myString.find("World"); pos != std::string::npos) {
    std::cout << pos << "World\n";
}

The pos variable is limited to the if/switch statement scope.

Capturing [*this] in Lambda Expressions

C++17 added [*this]

This captures a copy of the whole object, not just the pointer.

Example:

autolam = [*this] {
    std::cout<<s<<'\n';
};

Now the lambda owns its own copied Baz object.

Conceptually:

structLambda {
Baz copied_object;

voidoperator()()const {
        std::cout<<copied_object.s<<'\n';
    }
};

So this becomes safe:

autof1 =Baz{"ala"}.foo();
f1();

because the lambda has its own copy of Baz.

Deduction Guides

Deduction guides are the rules used by CTAD to decide which template arguments should be used when we write a class template without explicitly writing its template parameters.

std::arrayarr{1,2,3,4};

Here we did not write:

std::array<int,4>arr{1,2,3,4};

Still, in C++17 the compiler deduces:

std::array<int,4>

This is possible because std::array has a deduction guide.

Syntax of Deduction Guide

The general syntax is:

template<typename T>
ClassName(parameters) ->ClassName<deduced_template_arguments>;

For std::array, a simplified deduction guide looks like this:

template<typename T,typename...U>
array(T,U...) ->array<T,1+sizeof...(U)>;

This means:

std::array arr{1,2,3,4};

The compiler sees T =int U... =int,int,int , therefore 1+sizeof...(U) =4

So the deduced type becomes:

std::array<int,4>

Important Point

A deduction guide is not a constructor.

It only tells the compiler:

“From these constructor arguments, deduce this class template specialization.”

So this:

std::array arr{1,2,3,4};

is treated as:

std::array<int,4> arr{1,2,3,4};

The deduction guide only helps in deciding:

T =int
N =4

Why std::array Needs Deduction Guide

std::array is roughly:

template<typename T, std::size_tN>
	struct array {
	T elems[N];
};

The problem is that N is not directly passed as a constructor argument.

So from:

std::array arr{1,2,3,4};

the compiler must somehow understand:

T =int
N =4

The deduction guide gives this information.

Custom Deduction Guide

We can also write our own deduction guides.

#include <string>

template<typename T>
	struct MyType {
	T str;
};

MyType(constchar*) ->MyType<std::string>;

Now MyTypet{"Hello World"} .Without the custom deduction guide, the compiler may deduce MyType<constchar*> but because we wrote MyType(constchar*) ->MyType<std::string>; the compiler deduces MyType<std::string> So MyTypet{"Hello World"}; becomes MyType<std::string> t{"Hello World"};

Fold Expressions

Before C++17, variadic template packs were usually processed using recursive templates.

Before C++17:

template<typename T>
	auto sum(Tt) {
	return t;
}

template<typename T1,typename...T>
	auto sum(T1s,T...ts) {
	return s+sum(ts...);
}

For:

autovalue =sum(1,2,3,4);

the compiler conceptually does:

1+sum(2,3,4)
1+2+sum(3,4)
1+2+3+sum(4)
1+2+3+4

This works, but it needs a base-case overload.

C++17 lets us write this directly:

template<typename...Args>
	auto sum(Args...args) {
	return (args+ ...);
}

Now:

auto value = sum(1,2,3,4);

expands roughly as:

1+ (2+ (3+4))

This is called a fold expression.

It means:

Apply one binary operator repeatedly over all elements of a parameter pack.

Four Types of Fold Expressions

Assume the pack is:

args =a1,a2,a3

1. Unary right fold

(args+ ...)

Expands to:

a1+ (a2+a3)

2. Unary left fold

(...+args)

Expands to:

(a1+a2)+a3

3. Binary right fold

(args+ ...+init)

Expands to:

a1+ (a2+ (a3+init))

4. Binary left fold

(init+ ...+args)

Expands to:

((init+a1)+a2)+a3

Parsing ints From the command line:

#include <iostream>
#include <optional>
#include <string>

std::optional<int> ParseInt(const char *arg) {
  try {
    return {std::stoi(std::string(arg))};
  } catch (...) {
    std::cerr << "cannot convert '" << arg << "' to int \n ";
  }

  return {};
}

int main(int argc, char *argv[]) {
  if (argc >= 3) {
    auto oFirst = ParseInt(argv[1]);
    auto oSecond = ParseInt(argv[2]);

    if (oFirst && oSecond) {
      std::cout << "sum of " << *oFirst << " and " << *oSecond;
      std::cout << " is " << *oFirst + *oSecond << "\n";
    }
  }
  return 0;
}
