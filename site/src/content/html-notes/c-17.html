Gauranteed Copy Elision

Optimisation method by which the compiler must construct certain temporary objects in their target storage omitting copy/move construction entirely.

This optimisation technique is so important that is works in -O0 also. To stop use the flag -fno-elide-constructors 

#include <iostream>
#include <utility>

struct S {
  int x;

  explicit S(int x) : x{x} { std::cout << "construct S(" << x << ")\n"; }

  S(const S &other) : x{other.x} { std::cout << "copy S(" << x << ")\n"; }
};

S make_value(int x) { return S(x); }

S make_double_value(int x) {
  S s = make_value(x);
  s.x = 2 * x;
  return s;
}

int main(int argc, char *argv[]) {
  S s = make_double_value(21);
  std::cout << "answer: " << s.x << "\n";
  return 0;
}
  • Output with -fno-elide-constructors flag:
❱❱❱ g++ GCE.cpp -fno-elide-constructors -O0 -std=c++11 && ./a.out
construct S(21)

copy S(21)
copy S(42)
copy S(42)
answer: 42
  • Output without:
❱❱❱ g++ GCE.cpp -O0 -std=c++17 && ./a.out
construct S(21)
answer: 42

Therefore we should only use the constructor to only contruct because other functionality might be omitted as in this case with the std::cout statement. Also avoid writing custom contructor because it cannot be subject to the same optimization techniques. 

#include <array>
#include <iostream>

struct NonMoveable {
  NonMoveable(int x) : v(x) {}
  NonMoveable(const NonMoveable &) = delete;
  NonMoveable(NonMoveable &&) = delete;

  std::array<int, 1024> arr;
  int v;
};

NonMoveable make(int val) {
  if (val > 0)
    return NonMoveable(val);
  return NonMoveable(-val);
}

int main(int argc, char *argv[]) {
  auto largeNonMoveableObj = make(90);
  return largeNonMoveableObj.v;
}

With C++17 we get clear rules on when elision is about to happen , in fact instead of eliding the copies the compiler defers the materialisation of an object.
This can be helpful in case of non copyable or non moveable objects o to improve code protability.

Updated Value Categories

Realted with Copy Elision, C++17 gave proposed defenitions updates for glvalue and prvalue:

  1. glvalue : expression whose evaluation computes the location of an object, bit-feild, or function
  1. prvalue : expression whose evaluation initialises an object, bit-feild, or function, or operand of an opeartor.

In C++17 Copy Elision/deffered tempoaray Materialization happens when:

  1. in initialisation of an object from a prvalue: Type t = T()
  1. in a function call where the function returns a prvalue

C++98/03 had only lvalue and rvalue, C++11 extended to xvalue, prvalue and glvalue

Dynamic Memory Allocation for Over-Aligned Data

We can use alignas keyword to over-align but as so C++11/14 there was no guarantee that memory will be aligned after new[] , often wee had to run routines like std::aligned_alloc() or _aliged_malloc() for alignment preservation. But with C++17 :

void* operator new(size_t, align_val_t);
void operator delete(void*, size_t, align_val_t);
